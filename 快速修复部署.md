# 快速修复 GitHub Pages 部署

## 🚨 问题：样式丢失

GitHub Pages 部署后样式丢失，页面布局混乱。

## ✅ 解决方案（1分钟）

### 步骤 1: 确认你的网站 URL

打开你的 GitHub 仓库，查看网站地址是哪种：

**A. 项目页面（最常见）**
```
https://username.github.io/仓库名
例如：https://zhangsan.github.io/xiaodu-blog
```
👉 需要配置仓库名

**B. 用户页面**
```
https://username.github.io
```
👉 不需要配置

**C. 自定义域名**
```
https://yourdomain.com
```
👉 不需要配置

### 步骤 2: 修改配置

打开 `next.config.ts`，找到这一行：

```typescript
const REPO_NAME = ''; // 👈 在这里填写你的仓库名
```

**如果是情况 A（项目页面）：**
改成你的仓库名，例如：
```typescript
const REPO_NAME = 'xiaodu-blog';
```

**如果是情况 B 或 C：**
保持为空字符串：
```typescript
const REPO_NAME = '';
```

### 步骤 3: 重新部署

```bash
npm run build
git add .
git commit -m "Fix deployment path"
git push
```

等待 2-3 分钟，GitHub Actions 会自动部署。

### 步骤 4: 验证

访问你的网站，强制刷新（Ctrl+Shift+R），检查样式是否正常。

## 📝 示例

### 示例 1: 仓库名是 `my-blog`
```typescript
const REPO_NAME = 'my-blog';
```

### 示例 2: 仓库名是 `xiaodu-blog`
```typescript
const REPO_NAME = 'xiaodu-blog';
```

### 示例 3: 使用自定义域名
```typescript
const REPO_NAME = '';
```

## 💡 优势

这个配置的好处：
- ✅ 本地开发：`http://localhost:3000` （无需加仓库名）
- ✅ 生产部署：自动添加仓库名路径
- ✅ 一次配置，永久有效

## 🔍 如何查看仓库名

1. 打开你的 GitHub 仓库
2. 查看页面顶部的仓库名
3. 例如：`username/xiaodu-blog` 中的 `xiaodu-blog` 就是仓库名

## ⚠️ 注意

- 仓库名区分大小写
- 不要包含斜杠 `/`
- 不要包含 `https://` 等前缀

## ✅ 完成！

修改后提交代码，等待自动部署完成即可。
