# 文章更新和部署指南

## 🎉 恭喜！你的博客已成功部署

## 📝 日常写作流程

### 方式 1: 本地写作（推荐）

#### 步骤 1: 创建新文章

在 `_posts` 目录下创建新的 `.md` 文件：

```bash
# 文件名格式：任意英文名.md
# 例如：my-new-post.md
```

**文章模板：**

```markdown
---
title: '文章标题'
date: '2024-01-26'
category: '分类名称'
excerpt: '文章摘要，会显示在列表页'
---

# 文章标题

这里是文章正文内容...

## 二级标题

更多内容...

### 三级标题

```javascript
// 代码示例
console.log('Hello, World!');
```

- 列表项 1
- 列表项 2

**粗体文字** 和 *斜体文字*
```

#### 步骤 2: 本地预览

```bash
# 启动开发服务器
npm run dev

# 访问 http://localhost:3000 查看效果
```

#### 步骤 3: 提交并推送

```bash
# 1. 添加所有更改
git add .

# 2. 提交（写清楚做了什么）
git commit -m "Add new post: 文章标题"

# 3. 推送到 GitHub
git push origin main
```

#### 步骤 4: 等待自动部署

- GitHub Actions 会自动开始构建（约 2-3 分钟）
- 访问你的域名查看新文章
- 如果没看到，强制刷新浏览器（Ctrl+Shift+R）

### 方式 2: 直接在 GitHub 上编辑

#### 步骤 1: 访问 GitHub 仓库

打开：`https://github.com/littlestronger666/my_blog`

#### 步骤 2: 创建新文章

1. 进入 `_posts` 目录
2. 点击 **Add file** → **Create new file**
3. 文件名：`my-new-post.md`
4. 填写内容（使用上面的模板）
5. 点击 **Commit changes**

#### 步骤 3: 等待部署

- GitHub Actions 自动构建
- 2-3 分钟后访问你的网站

## 📝 文章 Frontmatter 说明

每篇文章开头必须包含 frontmatter（元数据）：

```markdown
---
title: '文章标题'          # 必需：显示在页面上的标题
date: '2024-01-26'        # 必需：发布日期，格式 YYYY-MM-DD
category: '技术'          # 必需：文章分类
excerpt: '文章摘要'       # 可选：显示在列表页的摘要
---
```

### 示例

```markdown
---
title: '如何使用 Next.js 构建博客'
date: '2024-01-26'
category: '技术'
excerpt: '本文介绍如何使用 Next.js 和 Markdown 构建一个现代化的静态博客。'
---

# 如何使用 Next.js 构建博客

这是文章正文...
```

## 💡 写作技巧

### 文件命名建议

```
good-article-name.md        ✅ 推荐：小写字母，连字符分隔
my-new-post-2024.md         ✅ 推荐：包含日期
技术文章.md                  ⚠️ 可以，但 URL 会是编码后的
My Article.md               ❌ 不推荐：有空格
```

### 分类建议

保持分类简洁和一致：
- 技术
- 生活
- 随笔
- 教程
- 笔记

### 日期格式

必须使用 `YYYY-MM-DD` 格式：
- ✅ `2024-01-26`
- ✅ `2024-12-31`
- ❌ `2024/01/26`
- ❌ `01-26-2024`

## 🚨 常见问题

### Q1: 推送后网站没更新
**解决：**
1. 检查 GitHub Actions 是否成功
2. 等待 2-3 分钟
3. 强制刷新浏览器（Ctrl+Shift+R）
4. 清除浏览器缓存

### Q2: 文章不显示
**原因：**
- Frontmatter 格式错误
- 日期格式不正确
- 文件不在 `_posts` 目录

**解决：**
1. 检查 frontmatter 格式
2. 确认日期格式为 `YYYY-MM-DD`
3. 确认文件在正确的目录

### Q3: 构建失败
**解决：**
1. 查看 GitHub Actions 的错误日志
2. 检查 Markdown 语法是否正确
3. 确认所有必需字段都填写了

### Q4: 图片不显示
**解决：**
1. 将图片放在 `public` 目录
2. 使用相对路径：`/images/photo.jpg`
3. 或使用外部图床链接
