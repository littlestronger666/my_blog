## 🎯 日常工作流程

### 发布新文章

```bash
# 1. 在 _posts 目录创建文件
touch _posts/my-new-article.md

# 2. 编辑文件（使用文章模板.md）

# 3. 提交推送
```

## 💡 最佳实践

### 写作

1. **使用有意义的文件名**
   - ✅ `how-to-use-nextjs.md`
   - ❌ `article1.md`

2. **保持分类简洁**
   - 技术、生活、随笔、教程等
   - 不要创建太多分类

3. **填写完整的 frontmatter**
   - 标题、日期、分类都要填写
   - 添加摘要提高可读性

4. **本地预览后再发布**
   ```bash
   npm run dev
   # 访问 http://localhost:3000 预览
   ```

### Git 提交

1. **有意义的提交信息**
   - ✅ `Add: 如何使用 Next.js`
   - ✅ `Update: 修正文章中的错误`
   - ❌ `update`

2. **定期提交**
   - 写完一篇文章就提交
   - 不要积累太多更改

3. **推送前检查**
   ```bash
   git status  # 查看更改
   git diff    # 查看具体修改
   ```

### 维护

1. **定期备份**
   - 定期克隆仓库到本地
   - 或使用 GitHub 的导出功能

2. **监控部署**
   - 推送后检查 GitHub Actions
   - 确认部署成功

3. **更新依赖**（可选）
   ```bash
   npm update
   ```

## 🆘 常见问题

### Q: 推送后网站没更新？
**A:** 
1. 检查 GitHub Actions 是否成功
2. 等待 2-3 分钟
3. 强制刷新浏览器（Ctrl+Shift+R）

### Q: 文章不显示？
**A:**
1. 检查 frontmatter 格式
2. 确认日期格式为 `YYYY-MM-DD`
3. 确认文件在 `_posts` 目录

### Q: 如何添加图片？
**A:**
1. 将图片放在 `public/images` 目录
2. 在文章中使用：`![描述](/images/photo.jpg)`

### Q: 如何修改网站标题？
**A:**
编辑 `app/layout.tsx` 中的 metadata

### Q: 如何更改主题颜色？
**A:**
编辑 `tailwind.config.js` 和组件中的颜色类
