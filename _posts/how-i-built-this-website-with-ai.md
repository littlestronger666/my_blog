这是我的第一篇博客文章。这个网站我用一下午的时间用AI帮我进行构建并调试，最后通过GitHub部署上线，这篇文章我将详细介绍我是如何用AI构建并部署上线的。

## 技术背景

我是一名非计算机专业人士，之前的专业是土木工程，我对计算机一直比较有兴趣，只是因为本职工作比较忙，一直没有精力学习，后面换了份较轻松的工作之后，开始有一些时间去学习一些计算机方面的知识，目前对Python、前端语言（HTML、CSS、JavaScript）、docker、sqlite有一些初步了解。

## 技术栈

- Next.js - React 框架
- Tailwind CSS - 样式框架
- Markdown - 内容格式
- GitHub Pages - 部署平台

## 所用到的AI工具

- 谷歌NotebookLM
- gemini
- Kiro

## 构建过程

### 1. 生成提示词

这里我用的是谷歌的NotebookLM，它是基于Gemini模型根据用户上传的文档（PDF、视频等），生成带引用的摘要、见解和学习工具。他的核心定位是提供高准确性、确保隐私的个性化知识管理。下面是NotebookLM的界面，文章的来源我用的是技术博主'向阳乔木'的提示词文档，关于NotebookLM，我后面会专门出一篇文章介绍使用方法，这里暂不展开了。

![NotebookLM界面](./images/Pasted%20image%2020251117103819.png)

由于我是非程序员和产品经理出身，对于产品的需求非常迷糊，我需要一段提示词，让AI将我的一个迷糊的创意变成一份具体需求的文档，这样agent工具才能根据这份需求文档生成比较好的结果，这里我通过输入"我想制作一个个人网页，我需要你生成一个高质量的提示词，用来将我的模糊的创意变成可落地的方案"，NotebookLM给我生成的提示词如下：

![生成的提示词](./images/Pasted%20image%2020251117105508.png)

### 2. 生成需求文档

我将生成的提示词复制到gemini，gemini将根据你提供的提示词，扮演一个AI架构师的角色，此时gemini会让你描述一个您想要解决的问题、一个系统愿景，或者一个尚未成型的初步想法。AI将开始解码其核心价值与潜力，并准备好提出深入的问题以完善它。

![Gemini界面](./images/Pasted%20image%2020251117105724.png)

再次输入我的想法："我想要做一个个人的博客网页，请生产一份可以直接让AI完成的需求文档"，gemini会根据我的想法生成一份详细的需求文档，并给出下一步的行动建议。

![需求文档生成](./images/Pasted%20image%2020251117110031.png)

![行动建议](./images/Pasted%20image%2020251117110205.png)

根据自己的需求确定其中的选项后，gemini会生成详细的需求文档：

![详细需求文档1](./images/Pasted%20image%2020251117110409.png)

![详细需求文档2](./images/Pasted%20image%2020251117110712.png)

![详细需求文档3](./images/Pasted%20image%2020251117110733.png)

### 3. Kiro根据需求文档生成网页框架

这里我选择Kiro作为coding的工具，他是亚马逊Agentic IDE，专注于规格驱动开发（SDD），将需求分解为结构化设计和任务。它通过智能体挂钩自动化测试和文档等后台任务，旨在提高代码质量和工程治理，而非单纯的速度。其实一开始我用的是字节出的Trea，但是这款coding工具目前来看代码能力一般，后来在某个群里发现Kiro这个软件，这个软件自带Claude，目前免费版本对于普通的开发者是够用的，Kiro内置最新的Claude sonnet4.5和Claude Haiku 4.5，代码质量是比较高的。

新建一个项目文件夹，将上面的需求文档保存一份放到这个文件夹内，软件打开项目文件夹后，输入"请根据文件夹中的需求文档，帮我建立个人网站"，软件开始根据需求文档工作，自动完成相应代码的编写。

![Kiro工作界面1](./images/Pasted%20image%2020251117112002.png)

![Kiro工作界面2](./images/Pasted%20image%2020251117112111.png)

### 4. 本地调试

Kiro完成所有代码工作后，会给出具体的说明文档，包括本地调试和GitHub线上部署的具体步骤，非常详细，对小白非常友好，通过本地调试，发现了亮/暗模式切换失败，我把问题喂给Kiro，也都顺利帮我解决了。

### 5. GitHub部署

一个网页从无到有，再到线上部署，最后这个过程是非常让人兴奋的，Kiro在生产的使用说明的文档中，非常具体的给出了如何在通过GitHub Pages部署的步骤，虽然第一次通过CLI界面对代码进行上传，但由于步骤详细，中间也没有出现大的问题，另外我强烈建议大家都去Dpdns注意一个免费的域名（后面有时间，我也会出一个教材），然后通过绑定域名，你就有了自己域名的网页啦，另外评论系统Kiro也给了具体的操作步骤，最后也顺利加入到网页了。

这是网页最后呈现的效果（亮/暗模式）🎉🎉🎉🎉，虽然看起来比较简单，到我觉得挺简洁的，以后这里就是属于我自己的一个小天地了，哈哈
